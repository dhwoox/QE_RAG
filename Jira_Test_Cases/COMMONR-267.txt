■ 이슈 키: COMMONR-267
■ 제목: 네트워크 연결된 상태에서 방치에 따른 장치 Reboot 유무 확인 케이스
============================================================

■ 테스트 스텝

[Step 1]
스텝 (Step)
1. 단일장치에 대해 전원인가 및 네트워크 연결된 상태
2. 장치의 RS485 모드 설정
3. 3주 방치
4. Reboot 유무 확인

데이터 (Data)
RS485 모드
: Default

예상 결과 (Expected Result)
1. 장치 Reboot 발생되지 않아야 한다.
------------------------------------------------------------
[Step 2]
스텝 (Step)
1. RS485 구성으로 연결
2. 3주간 방치
3. Master/Slave장치에 대해 Reboot 유무 확인

데이터 (Data)
N/A

예상 결과 (Expected Result)
1. 장치 Reboot 발생되지 않아야 한다.
------------------------------------------------------------
[Step 3]
스텝 (Step)
1. RS485 구성으로 연결
2. Slave와 연결된 RS485 케이블 물리적으로 제거
3. 3주간 방치
4. Master/Slave장치에 대해 Reboot 유무 확인

데이터 (Data)
N/A

예상 결과 (Expected Result)
1. 장치 Reboot 발생되지 않아야 한다.
------------------------------------------------------------
