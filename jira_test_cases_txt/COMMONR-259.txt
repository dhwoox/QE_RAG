■ 이슈 키: COMMONR-259
■ 제목: Device ID 변경 지원 - 재기획필요
■ 설명: 요구 사항
 # 신규 장치 ID의 변경은 master와 slave 구분 없이 모두 설정할 수 있어야 한다. RS-485 로 연결된 slave 장치의 신규 장치 ID 변경은 별도의 프로토콜로 지원한다.
 # 신규 장치 ID의 변경은 리더 장치뿐만이 아닌 SIO2, DM-20, OM-120 등의 악세서리 장치들도 지원되어야 한다.
 # 신규 장치 ID의 표현 제약은 기존 시리얼 번호 기반의 장치 ID 제약을 그대로 따른다.
 -- Device ID 지원 범위 - 16777216 ~ 0x3FFFFFFF(=1073741823) 지원한다.(CS40의 경우 최대 0x30FF FFFF(=822083583) 지원)
 -- 장치에서 지원하지 않는 Device ID 체계로 변경시도가 될 경우 Error를 출력한다.
 # 신규 장치 ID의 초기값은 해당 장치의 시리얼 번호를 따른다.
 # 기존 시리일 번호는 BS2SystemConfig 의 reserved 영역 중 4 byte를 사용하여 백업한다.
 -- 기존 시리얼 번호를 저장하는 이유는 최초 시리얼 번호를 추적할 수 있기 위함이다.
 # 신규 장치 ID를 변경할 경우, BS2_CMD_SET_DEVICE_ID 프로토콜을 이용하여 실제 장치 ID를 변경한다.
 # Factory Default 시 현재 저장된 신규 장치 ID는 유지된다.
 # 신규 장치 ID 설정 시 기존 설정, 사용자, 로그 등 모든 정보를 factory reset 하여 삭제한다.
 -- 이때, BS2IpConfig, BS2IpConfigExt 는 초기화하지 않는다. 이는 장치 ID를 변경하기 위해 미리 네트워크 설정이 된 장치들에 대해 초기화 이후 다시 설정하지 않도록 하기 위함이다.(2020.04.14)
 -- 해당 초기화 동작은 기존의 Factory Reset 로그가 아닌 Factory Reset without Network 라는 신규 로그를 남기도록 한다.(2020.04.20)
 # Slave 장치를 지니는 Master 장치 교체시 RS485 통신에 사용되는 키가 달라짐에 따라 연결이 되지 않는 문제는 BioStar 2 에서 일괄적으로 장치의 RS485 통신 key 를 관리하고 전달하는 방식으로 해결한다. 이는 BioStar 2 의 기획에 포함된다.
 # 장치 ID 변경이 지원되지 않는 버전으로 FW 다운그레이드할 경우 SystemConfig 에 백업해놓은 원본 시리얼로 장치 ID 를 변경 후 남은 다운그레이드 프로세스를 진행하도록 한다.
 # LCD 지원장치의 경우 Device Info에 변경ID와 기존ID를 모두 표시한다.
 -- Device ID(장치 ID) 와 Serial No(시리얼번호)로 표시한다.(영문/국문)
 # Wiegand Reader에 대해서는 Device ID 변경을 미지원한다.
 # Device ID 변경시 Mac Address는 원본 Mac Address를 본다.
============================================================

[Step 1]
스텝 (Step)
1. BioStar2에서 장치 추가
2. Device ID 변경할 장치에서 우클릭
3. "장치ID 변경" 옵션 클릭
4. Device ID 변경시도 후 동작확인

데이터 (Data)
변경할 장치ID
: 9자리의 영숫자
: 9자리의 숫자+공백
: 3byte(=16777216) 숫자
: 4byte(=1073741823, CS40 - 822083583) 외 임의값
