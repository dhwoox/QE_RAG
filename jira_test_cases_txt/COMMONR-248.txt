■ 이슈 키: COMMONR-248
■ 제목: 펌웨어 복구(Recovery) 기능 검증TC - 미지원 기능
■ 설명: > FW Upgrade 중 전원이 재인가된 경우 이전 버전으로의 복구여부 확인
> 잘못된 베타FW 등이 Upgrade된 경우 이전 버전으로의 복구여부 확인

※ 발생가능케이스는 계속 업데이트 예정


확인방법
Recovery 기능 포함된 FW로 업그레이드된 상태에서 검증진행(FW2_2019 1H 릴리즈 버전부터 지원)
> 처음으로 Upgrade한 Recovery 포함된 버전으로 백업된다.(= Backup 파일이 존재할 경우 다시 백업 하지않음)


해당기능 삭제.(2020 1H Release 버전부터 제외)



============================================================

■ 테스트 스텝

[Step 1]
스텝 (Step)
1. FW Upgrade 시도
2. Upgrade 진행 중 장치의 전원 재인가
   > 장치에 Write 중일 때
3. 부팅 후 FW 정보 확인

데이터 (Data)
장치모드: Default/Master

예상 결과 (Expected Result)
1. 전원인가 후 나머지 부분에 대해 Write된다.

스텝 (Step)
<RS485 구성>
1. Slave 장치에 FW Upgrade 시도
2. Slave Upgrade 진행 중 Slave 장치의 전원 재인가
   > 장치에 Write 중일 때
3. 부팅 후 FW 정보 확인

데이터 (Data)
N/A

예상 결과 (Expected Result)
1. 전원인가 후 나머지 부분에 대해 Write된다.
------------------------------------------------------------

스텝 (Step)
1. 잘못된 FW Upgrade 시도
