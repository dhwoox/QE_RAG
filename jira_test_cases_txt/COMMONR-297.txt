■ 이슈 키: COMMONR-297
■ 제목: [FISF-2137] 중복얼굴 처리에 대한 재발방지대책
■ 설명: FISF-2137


환경구성

RS485 구성
 => F2(M)-F2(S) or BS3(M)-BS3(S)

Door 구성
 > Entry Device: Master Device
 > Exit Device: Slave Device
 > Relay: Master Device
============================================================

■ 테스트 스텝

[Step 1]
스텝 (Step)
1. 바이오스타 > 동일한 사진으로 2명 이상의 사용자에 Visual Face 등록 후 장치로 전송
2. 중복으로 등록된 사용자로 인증시도
3. 중복으로 등록되지 않은 사용자로 인증시도
4. 결과출력확인

데이터 (Data)
Face 인증모드
> Face Only
> Face + Finger
> Face + PIN
> Face + Finger/PIN
> Face + Finger + PIN

예상 결과 (Expected Result)
1. 중복사용자로 인증시 실패가 출력되어야 한다.
2. 중복으로 등록되지 않은 사용자로 인증시 Auth Mode Error가 표시되지 않아야 한다.
------------------------------------------------------------
[Step 2]
스텝 (Step)
[중복얼굴 등록 후 장치로 전송]
1. 중복으로 등록된 사용자로 인증시도
2. 중복으로 등록된 사용자 1초 정도 카메라에 노출 후 중복으로 등록되지 않은 사용자로 인증시도
3. 결과확인

데이터 (Data)
Face 인증모드
> Face Only
> Face + Finger
> Face + PIN
> Face + Finger/PIN
> Face + Finger + PIN

인증장치
> Master
> Slave

예상 결과 (Expected Result)
1. 중복사용자로 인증시 실패가 출력되어야 한다.
2. 중복으로 등록되지 않은 사용자에 대해 인증성공으로 동작되고 Relay가 동작되어야 한다.
------------------------------------------------------------
