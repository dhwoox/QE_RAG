■ 이슈 키: COMMONR-247
■ 제목: [BSPP-412] Batch Edit에서 장치 Admin 추가방식 변경관련 검증TC
■ 설명: 1. 장치 Batch Edit에서 장치 관리자 설정시 Append 로 변경하여 지원한다.
> FW버전 상관없이 기 설정된 관리자는 유지하고 새로 추가하고자 하는 관리자에 대해서만 추가(이전스펙: Delete&Set으로 동작됨)
> 신규 장치에 관리자가 설정되어 있고 일괄 편집으로 관리자를 추가 시 '기존 관리자 + 일괄편집으로 추가한 관리자'로 저장됨
> 신규 장치에 관리자가 설정되어 있고 일괄 편집으로 설정되어 있는 관리자를 다른 권한으로 변경 시 변경한 권한으로 저장 됨
> 일괄 편집으로 추가되는 관리자로 인해 장치의 최대 관리자 사용자 개수를 초과할 경우 에러 없이 최대 사용자까지 저장되고 초과되는 관리자는 저장되지 않음 (고객이 원하는 관리자를 설정할 수 없는 문제는 known issue, 추후 개선 필요)
> 일괄 편집으로 관리자 삭제는 불가 함 (요구사항이 Append방식으로 변경하는 것으로 일괄편집으로 관리자를 삭제하거나 Overwrite를 할 수 없음, 추후 개선이 필요 함)

2. Append 는 최신 FW만 동작 
> 2019 1H에 릴리즈되는 버전부터 적용

3. 기존 FW는 기존과 동일하게 Delete & Set으로 지원 
> ~ 2018 2H에 릴리즈된 버전

4. 초기화시 관리자 Config에 대해 삭제한다.(기존과 동일)
============================================================

■ 테스트 스텝

[Step 1]
스텝 (Step)
1. 장치 두대이상 선택> Batch Edit 클릭
2. 다수의 관리자(All/User/Config) 최대로 설정 후 적용
3. 장치상세정보창 진입
4. 관리자 확인
5. All/User/Config 관리자로 메뉴진입시도

데이터 (Data)
> Max 관리자수: 1000명
> 각 장치별 관리자 미설정된 상태

예상 결과 (Expected Result)
1. All/User/Config 관리자가 설정된다.
: 장치상세정보창_관리자 항목에 표시
2. 관리자로 메뉴진입시 각 관리자에 해당하는 메뉴에 접근이 가능해야 한다.
