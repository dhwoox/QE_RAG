■ 이슈 키: COMMONR-327
■ 제목: [BDP-805] Intelligent Slave 사용 시 CP 로부터 전송받은 인증 결과(osdp_LED)를 장치 화면에 표시하기
■ 설명: https://jira.suprema.co.kr/browse/BDP-805

1.요구사항
 * 1.1 액정이 탑재된 슈프리마 장치를 Intelligent Slave 또는 3rd Party CP 에 대해 PD 로 사용할 때, 3rd Party CP 로부터 받는 LED 커맨드를 통해 CP 의 인증결과를 장치의 액정과 스피커로 출력한다.
 ** 아래의 경우에만 인증 결과에 대한 LED 출력 커맨드로 간주한다. 
 *** 카드 값을 보낸 후 3초 내로 받는 osdp_LED 커맨드
 ** CP 로부터 RED osdp_LED 커맨드를 수신하면 장치 화면에 "Access Denied" UI 와 소리를 출력한다.
 ** CP 로부터 GREEN osdp_LED 커맨드를 수신하면 장치 화면에 "Access Granted" UI 와 소리를 출력한다.
 ** 이 때, 장치에서의 인증 결과 UI 는 인증 실패만 표시하고, 인증 성공 UI 는 표시하지 않는다.
 *** 이는, 인증 성공 시, CP 로부터 또 인증 피드백을 받기 때문에 장치의 인증 결과와 CP 의 인증 결과가 중복으로 나타나므로 UI 가 부자연스러워지기 때문이다.
 * 1.2 액정과 LED 둘 다 탑재된 장치 또한, 해당 동작을 수행한다.
 ** 액정과 LED 둘 다 있는 경우 사용자는 액정을 주된 인터페이스로 인지하므로 LED 가 있다고 하더라도 액정에 피드백을 표시하도록 한다.
 * 1.3 Master 가 타사 장치인 경우에만 해당 설정이 적용된다.
 ** 즉, Intelligent Slave 또는 3rd Party CP 에 대해 PD 로 사용될 때를 의미한다.
 * 1.4 해당 기능은 옵션으로 제공한다.
 ** 기본값 : 1. CP의 결과 표시
 ** 기능명
 *** 국문 : 인증 결과
 *** 영문 : Authentication Result
 ** 옵션명
 *** 1. CP의 결과 표시
 **** 국문 : 컨트롤러 결과 표시
 **** 영문 : Display Result from Controller
 *** 2. 장치의 결과 표시
 **** 국문 : 장치 인증 결과 표시
 **** 영문 : Display Device Matching Result
 * 1.5 해당 설정은 장치 및 BioStar 2 에서 접근할 수 있다.
 ** 장치
 *** Network > Serial Setting (OSDP Setting 으로 변경) 하위
 **** 단, RS-485 모드가 Default 또는 Slave 일 때만 해당 설정이 나타나야 한다.
 ** BioStar 2
 *** 기능 위치 : Network > Serial(OSDP 로 변경) 하위
 * 1.6 적용 모델 : BS3, FSF2, XS2, N2, *BSA2, BS2A, FS2(TBD)*
 * 1.7 Capability
 ** 해당 기능 지원 유무를 Capability 에 추가한다.

 검증환경
 > CS40 V1.6.1 이상버전 사용(3rd party OSDP 리더 지원 버전)
 > OSDP Device로 연결할 장치 모드 - Default/Slave
 > CS40에서 OSDP Reader LED/Buzzer 설정 - Auth Success(=Green) & Auth Fail(=Red)

환경설정
 > CS40 RS485 Port 에 Suprema 장치 연결
 > CS40에서 마우스 우클릭해 Add OSDP Device 클릭
 > OSDP Reader 선택 후 OK
 > 장치가 연결된 Port 선택
 > 연결된 장치의 OSDP ID와 동일하게 설정 후 Apply
 > 통신이 되는 상태에서 TC 진행

※ Slave 모드일 때의 환경설정
 Default 상태로 CS40과 통신중인 상태에서 RS485 모드를 Slave로 변경해서 진행

 

※ Station 계열들은 LED 상태 변경을 지원하지 않기 때문에(바이오스타에서 LED/Buzzer 설정 미지원) Default 값인 Green(인증성공), Red(인증실패)로 동작함.

따라서 CS40/CS20-Station 계열을 연결해 확인할 경우 CS40/CS20에서 OSDP 리더 LED 및 부저 설정시 인증성공: Green & 인증실패: Red 로 설정 후 검증 진행해야 함.

그렇지 않고 만약 CS40/CS20에서 LED를 Blue(인증성공), Magenta(인증실패)로 설정할 경우 Station 계열에 카드를 리드하더라도 아무 반응이 없을 것임(Display Result from Controller 옵션)
