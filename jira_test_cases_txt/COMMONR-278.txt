■ 이슈 키: COMMONR-278
■ 제목: [Intelligent Slave] Output Info : Card ID 사용
■ 설명: *요구사항 정의*
 > [http://192.168.1.4:8001/browse/BDP-92]

*관련 테스트를 진행하기 위한 전제조건*
 >CS40 V1.5.1 / BEW2 V1.6.3 / BSL2 V1.6.1 / BEP2 V1.4.1 / XP2 V1.2.3 이상 버전
 > RS-485 mode 가 Default 상태로 Suprema 장치가 아닌 3rd-party Controller 에 연결
 > Output info : Card ID 사용

*작성된 TestCase 내용*

> [출력정보: 카드ID] 카드인증 성공시 출력되는 값에 대한 확인케이스
 - CSN Card
 - Wiegand Card
 - Smart Card
 - Mobile Access Card

> [출력정보: Card ID] 카드인증 실패시 Exception Code 설정에 따라 출력되는 값에 대한 확인케이스

> [출력정보: Card ID] 카드 외 크리덴셜로 인증시 출력되는 값에 대한 확인케이스
 - 사용자에 카드가 미등록된 경우
 - 사용자에 카드가 등록된 경우

*HID OSDP Tool과 연결해 카드데이터를 출력시 동작*
 - 위겐드 카드를 리드한 경우 : ex. HID 26BIT 카드리드시 BitCount 26BIT로 출력
 - (Output Info: Card ID & User ID Type: 숫자인 경우) 위겐드 포맷으로 저장된 CSN 카드를 리드한 경우: Mifare 32bit 카드를 26bit 위겐드 포맷으로 저장 후 리드할 경우 BitCount 26bit로 출력
 - (Output Info: User ID & User ID Type: 숫자인 경우) 위겐드 포맷으로 저장된 CSN 카드를 리드한 경우: Mifare 32bit 카드를 26bit 위겐드 포맷으로 저장 후 리드할 경우 User ID가 출력되면서 User ID에 해당하는 BitCount 출력
 - (Output Info: Card ID & User ID Type: 영숫자인 경우) 위겐드 포맷으로 저장된 CSN 카드를 리드한 경우: Mifare 32bit 카드를 26bit 위겐드 포맷으로 저장 후 리드할 경우 BitCount 26bit로 출력
 - (Output Info: User ID & User ID Type: 영숫자인 경우)
 ==> 위겐드 카드의 경우 위겐드 포맷을 보고 해당 Format에 대해 BitCount로 출력시킴
 ==> CSN 카드의 경우 카드ID 전체를 들고 있어 내보낼 때 Wiegand Format으로 변환해 출력시킴

※ XP2의 경우 카드 리드뿐 아니라 Keypad 로 카드ID 입력해 동작되는 것까지 확인이 되어야 함.

 

*※ OSDP 표준에 대한 정상 동작 (BS3FW-1970)*
> 컨트롤러의 Intelligent Slave로 동작하는 장치들의 경우, Auth Mode에 맞는 Credential에 대한 창을 띄워주는 것이 아닌 사용자가 해당 Auth Mode에 맞게 Credential을 입력해주어야 함
============================================================
